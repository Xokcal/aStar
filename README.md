# i32-AStar-Pathfinding

一个用 C 语言从零手写的 A\* 寻路算法，包含曼哈顿距离、`f = g + h` 公式、堆数组管理、`parent` 指针链回溯，支持任意 20x20 迷宫。

```c
i32 manhattan_formula_exe(i32 start_x, i32 start_y, i32 t_x, i32 t_y) {
    return abs(start_x - t_x) + abs(start_y - t_y);
}

i32 enter_heap_formula(i32 curr_step, i32 manhattan_reuslt) {
    return curr_step + manhattan_reuslt;
}
```

## 核心理解

### A\* 的本质：从所有已知候选里挑最优

> 就是找到周围的四个节点，哪一个算出来的曼哈顿值最小就选哪个？不是。A\* 是把所有待探索的格子都放在一个列表里，每次从整个列表中，选出 `f = g + h` 最小的那个格子去探索。探索完这个格子，就把它的邻居加入列表。重复这个循环，直到找到终点。

**A\* 不是只看眼前，而是从所有已知的候选格子中，选出全局最优的那个。**

### `f = g + h` 是什么？

> `f` 是这个格子的权重，数值越小就越值得走。`g` 是从起点到当前格子一共走了多少步。`h` 是曼哈顿距离，估算从当前格子到终点最少还要走几步。

**`f` 越小，意味着“已经走得不多 + 离终点也近”，最值得优先探索。**

### 每个节点为什么要保存自己的 `step`？

> 如果所有格子共用一个全局步数，那不同路径的步数就会互相污染。A\* 会在不同路径之间来回跳跃，每个格子必须自己记住“从起点走到我这里需要几步”。

**每个节点自带独立步数，互不干扰。这就是为什么 `HeapNode` 里有 `step` 字段。**

### `parent` 指针链回溯路径

> 假如我走到了一步，下面和左边 f 值都一样，我走下面，但下面是死胡同。然后我在堆里发现上一步的左边是当前最小 f 值，我又走回上一步的左边。那这样 parent 不就乱了吗？等等好像没乱——上一步的左边的父坐标就是上一步，然后我回溯路径，直接就是上一步左边坐标到上一步的坐标，然后再到上上一步！

**每个节点的 parent 在被发现时就固定了，无论 A\* 之后怎么跳跃，历史记录都不会被篡改。回溯时沿着 parent 指针链一路跳回起点，路径完全正确。**
